    Class AudioSender

    This component represents a local audio source added as an audio track to an existing WebRTC peer connection and sent to the remote peer.

    WARNING

    Currently the low-level WebRTC implementation does not support registering local audio callbacks, therefore the audio track cannot be rendered locally with a MediaPlayer.

    RegisterCallback(AudioFrameDelegate)

    Register a frame callback to listen to outgoing audio data produced by this audio sender and sent to the remote peer.

    WARNING

    Currently the low-level WebRTC implementation does not support registering local audio callbacks, therefore this is not implemented and will throw a NotImplementedException.

    Declaration
    public void RegisterCallback(AudioFrameDelegate callback)
    Parameters
    Type Name Description
    AudioFrameDelegate callback

    The new frame callback to register.
    Remarks

    Unlike for video, where a typical application might display some local feedback of a local webcam recording, local audio feedback is rare, so this callback is not typically used. One possible use case would be to display some visual feedback, like an audio spectrum analyzer.

    Note that registering a callback does not influence the audio capture and sending to the remote peer, which occur whether or not a callback is registered.
