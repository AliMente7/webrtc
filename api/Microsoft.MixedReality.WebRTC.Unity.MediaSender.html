    AutoStartOnEnabled

    Automatically start media capture when the component is enabled, and stop capture when the component is disabled.

    If true, then StartCaptureAsync() is automatically called when the callback is invoked by Unity, and conversely StopCapture() is automatically called when the callback is invoked by Unity.

    If false, then the user has to manually call StartCaptureAsync() to start capture before the media sender is attached to a transceiver, and call StopCapture() to stop capture.

    Declaration
    public bool AutoStartOnEnabled
    Field Value
    Type Description
    Boolean
    Remarks

    When the media sender is attached to a transceiver, media capture starts automatically if the component is active and enabled, even if AutoStartOnEnabled is false.

    To have complete manual control on capture, set both AutoStartOnEnabled and to false.

    TrackName

    Name of the local media track this component will create when calling StartCaptureAsync(). If left empty, the implementation will generate a unique name for the track (generally a GUID).

    Declaration
    public string TrackName
    Field Value
    Type Description
    String
    Remarks

    This value must comply with the 'msid' attribute rules as defined in https://tools.ietf.org/html/draft-ietf-mmusic-msid-05#section-2, which in particular constraints the set of allowed characters to those allowed for a 'token' element as specified in https://tools.ietf.org/html/rfc4566#page-43:

    • Symbols [!#$%'*+-.^_`{|}~] and ampersand &
    • Alphanumerical characters [A-Za-z0-9]

    Users can manually test if a string is a valid SDP token with the utility method Validate(String, Boolean).

    Mute(Boolean)

    Mute or unmute the media. For audio, muting turns the source to silence. For video, this produces black frames. This is transparent to the SDP session and does not requite any renegotiation.
